## 🚀 Ключевой проект (показать работодателю в первую очередь)

### 📄 Итоговый командный проект: Потоковая обработка данных

**Файл:** `ИТОГОВЫЙ КОМАНДНЫЙ ПРОЕКТ_Потоковая_обработка данных_Иннополис.docx`

**Что это:**  
Сквозной проект, выполненный **в команде** (симуляция реальной рабочей среды). Задача — построить отказоустойчивый пайплайн для обработки потоковых данных в реальном времени.

**Архитектура решения (вероятно, включает):**
- **Источник:** симуляция событий / IoT / логи
- **Брокер сообщений:** Apache Kafka
- **Обработка:** Spark Streaming / Kafka Streams
- **Хранилище:** ClickHouse (для аналитики)
- **Оркестрация / мониторинг:** возможно, NiFi или контейнеры

**Демонстрируемые навыки:**
- Работа в команде над data-проектом (распределение задач, код-ревью, документация)
- Интеграция нескольких технологий в единый пайплайн
- Настройка кластеров Kafka и ClickHouse
- Обеспечение fault tolerance и exactly-once семантики
- Замер производительности и оптимизация

**Почему это важно для работодателя:**  
Показывает, что вы умеете **не только отдельные технологии, но и собирать из них работающую систему**, а также взаимодействовать в команде — ключевой софт-скилл для Data Engineer.

## 🔧 Другие лабораторные работы (база)

### 1. Apache Spark — пакетная обработка
**Файлы:** `SPARK_LAB_Гусев_К_А_.ipynb` + `SPARK_LAB _ Гусев К.А.pdf`

- Загрузка и трансформация больших данных
- Оптимизация запросов через партиционирование и кэширование
- Работа с DataFrame API и Spark SQL

### 2. Apache Kafka — брокер сообщений
**Файл:** `Работа с брокером_Kafka_Гусев К.А (3).pdf`

- Развертывание кластера
- Producers и consumers на Python
- Партиционирование, consumer groups
- Обработка dead letter queue

### 3. Apache NiFi — визуальный ETL
**Файл:** `Использование Apache_NIFI_Гусев К.А (3).pdf`

- Построение графа для ingesta данных
- Процессоры: `GetHTTP`, `EvaluateJsonPath`, `PutFile` и др.
- Управление backpressure и очередями

### 4. ClickHouse — колоночная аналитическая БД
**Файл:** `Работа с кластером ClickHouse_Кирилл Гусев (3).pdf`

- Установка и настройка кластера ClickHouse
- Вставка потоковых данных через Kafka Engine
- Написание аналитических запросов (агрегации, окна)
